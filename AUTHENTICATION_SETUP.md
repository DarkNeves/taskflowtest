# 🔐 Configuração de Autenticação - TaskFlow

Este documento guia você na configuração completa de autenticação com Supabase.

## ✅ O que foi implementado

- ✨ Sistema de Login/Sign Up com email e senha
- 🔒 Proteção de rotas (redireciona para login se não autenticado)
- 💾 Sessão persistente (mantém login ao atualizar página)
- 👤 Integração com tarefas (cada usuário tem suas próprias tarefas)
- 🎨 Design moderno seguindo o estilo visual do TaskFlow
- 📱 Interface responsiva (mobile + desktop)

## 🚀 Como configurar

### 1. Criar projeto no Supabase

1. Acesse https://supabase.com
2. Clique em "New Project"
3. Escolha um nome e uma senha para seu projeto
4. Aguarde o projeto ser criado

### 2. Obter credenciais

1. Vá para "Settings" → "API" no seu projeto Supabase
2. Copie:
   - `Project URL` (será seu `VITE_SUPABASE_URL`)
   - `anon public` key (será seu `VITE_SUPABASE_ANON_KEY`)

### 3. Configurar variáveis de ambiente

1. Copie o arquivo `.env.example` para `.env.local`:
   ```bash
   cp .env.example .env.local
   ```

2. Edite `.env.local` e adicione suas credenciais Supabase:
   ```
   VITE_SUPABASE_URL=https://seu-projeto.supabase.co
   VITE_SUPABASE_ANON_KEY=sua-chave-anonima
   ```

### 4. Criar tabela de autenticação (Opcional - Supabase cria automaticamente)

Supabase gerencia autenticação automaticamente. Mas se quiser sincronizar tarefas com o banco:

```sql
-- Na sua dashboard Supabase, vá para SQL Editor e execute:

CREATE TABLE tasks (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  nome TEXT NOT NULL,
  desc TEXT,
  data DATE NOT NULL,
  hora TIME,
  prio TEXT NOT NULL CHECK (prio IN ('alta', 'media', 'baixa')),
  done BOOLEAN DEFAULT false,
  done_at BIGINT,
  recurrence TEXT DEFAULT 'none',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  
  CONSTRAINT tasks_user_id_unique UNIQUE (user_id, id)
);

-- Row-level security (cada usuário vê só suas tarefas)
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can see their own tasks"
  ON tasks
  FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert their own tasks"
  ON tasks
  FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update their own tasks"
  ON tasks
  FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "Users can delete their own tasks"
  ON tasks
  FOR DELETE
  USING (auth.uid() = user_id);
```

## 🔄 Fluxo de Autenticação

1. **Usuário acessa o site**
   - App.tsx verifica se está autenticado
   - Se não: redireciona para `/login`
   - Se sim: mostra o dashboard

2. **Login/Sign Up**
   - Usuário entra email, senha (e nome para sign up)
   - Supabase autentica e retorna token
   - Sessão é armazenada em `localStorage`

3. **Persistência**
   - `onAuthStateChange` do Supabase
   - Se refresh da página: restaura sessão automaticamente

4. **Tarefas**
   - `useTaskStore` agora usa `user.id` como chave
   - Cada usuário tem suas tarefas no localStorage
   - Tarefas são isoladas por usuário

## 💡 Próximos Passos (Opcional)

### Sincronizar tarefas com Supabase (Banco de dados)

Se quiser que as tarefas sejam salvos também no banco de dados Supabase (não só localStorage):

```typescript
// Em client/src/hooks/useTaskStore.ts
// Adicionar sincronização com Supabase:

const syncTasks = async () => {
  if (!userId) return;
  const { data, error } = await supabase
    .from("tasks")
    .select("*")
    .eq("user_id", userId);
  
  if (error) console.error(error);
  if (data) setTasks(data);
};
```

### Email de verificação (recomendado)

Pode ativar verificação de email na dashboard do Supabase:
- Settings → Authentication → Email
- Ativar "Confirm email before signing in"

### Tema claro/escuro na autenticação

O login já respeita o tema do site automaticamente via `ThemeProvider`.

## 🔒 Segurança

- Senhas não são armazenadas (Supabase usa bcrypt)
- Tokens ficam em `localStorage` (seguro para web)
- `Row-level security` no banco garante isolamento
- Rotas protegidas impedem acesso direto

## ❓ FAQ

**P: As tarefas são salvas no Supabase ou localStorage?**
R: Por enquanto, localStorage. Pode sincronizar com Supabase opcionalmente.

**P: E se o usuário limpar o cache/cookies?**
R: Será feito logout, precisa fazer login novamente.

**P: Posso usar Google/GitHub para login?**
R: Sim! Supabase suporta OAuth. Vide docs: https://supabase.com/docs/guides/auth

**P: Qual é o limite de usuários?**
R: Supabase free tier: até 50.000 usuários.
